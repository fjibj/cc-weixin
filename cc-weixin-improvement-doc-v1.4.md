## 更新日志

### v1.4 (2026-04-05)

**核心改进：权限自动确认、进度通知、自动记忆**

#### 1. 权限自动确认机制 (P0)

**问题描述**:  
Claude 执行过程中遇到权限限制需人工确认，微信端无法感知导致等待。

**解决方案**:  
在 CLAUDE.md 中添加自动确认规则，无需人工干预即可执行常规操作。

**配置内容**:

| 操作类型 | 策略 | 说明 |
|----------|------|------|
| 文件创建/编辑 | ✅ 自动确认 | 工作空间内文件 |
| Bash 命令 | ✅ 自动确认 | 白名单内命令 |
| 子 Agent 创建 | ✅ 自动确认 | 最多 3 个并行 |
| 网络请求 | ✅ 自动确认 | API 调用、知识库上传 |

**Bash 命令白名单**:
```
git (add|commit|status|diff|log|push|pull)
npm (install|run|test|build|list)
bun (run|test|build|install|x)
node --eval
curl
mkdir|touch|cp|mv|rm|cat|echo
```

#### 2. 处理进度通知增强 (P2)

**问题描述**:  
长任务处理过程中用户无法感知进度，容易等待焦虑。

**解决方案**:
- 每 2 分钟发送一次进度更新
- 每个 Phase 完成时立即通知
- 通知格式标准化

**通知格式**:
```
【处理中】Step X/Y - 正在执行代码修改...
  当前文件：src/main.ts
  进度：40%
```

**实现效果**:
- 用户实时了解处理进度
- 避免"傻等"情况
- 提升用户体验

#### 3. 自动记忆机制 (P2)

**问题描述**:  
依靠手动记录到 MEMORY.md，容易遗漏。

**解决方案**:  
处理完成后自动记录到 `memory/weixin-history.md`

**记忆格式**:
```markdown
## {日期}
### 消息 N: {主题}
**消息**: {消息前 100 字}
**处理**: {流程简述}
**结果**: {结果摘要前 200 字}
**标签**: #{关键词}
```

**存储位置**:  
`memory/weixin-history.md`

**示例**:
```markdown
## 2026-04-05
### 消息 3: M-FLOW 技术调研
**消息**: "研究一下 M-FLOW 技术，说是超越了 RAG..."
**处理**: 多渠道搜索→GitHub 404→用户提供正确地址→完成调研
**结果**: M-FLOW=记忆增强知识图谱，Cone Graph 架构，基准领先 36%
**标签**: #调研 #M-FLOW #RAG #完成
```

#### 4. Harness 并行控制

**配置**:
```bash
/harness-work --parallel 3
```

**说明**: 最多 3 个子 Agent 并行处理，避免设备资源过载。
